title: Prompt Compression

**TraceCompress** is TraceGuard's prompt-and-output compression layer. It has two
jobs:

1. **Prompt compression** — make a messy prompt shorter and sharper without
   losing meaning or constraints.
2. **Output discipline** — generate strict response rules that tell the agent to
   produce minimal, direct, useful output.

It runs **locally and deterministically** by default — no model call, and no
prompt text leaves your machine. It never sends a compressed prompt to an agent
automatically: you always review and accept.

## What TraceCompress does

Prompt compression removes redundancy and filler:

- repeated instructions and duplicate requirements
- filler, emotional wording, useless politeness
- vague phrases and rambling explanations

…while **preserving** everything that changes behaviour:

- the task goal and acceptance criteria
- file names, folder names, commands, exact quoted strings
- API names, environment variables, URLs
- safety constraints, must-do and must-not-do instructions
- deployment, testing, UI, and architecture requirements

**Example**

Input:

```text
bro make the local dashboard better and like dont make it ugly and make sure it
doesnt add random features and dont use tauri and keep it localhost and like i
want the ui to be fancy but not cringe and also dont make the app lie about what
it did
```

Output (Concise): the filler ("bro", "like") is removed and the prompt is split
into clear clauses, while every constraint — *don't use Tauri*, *keep it
localhost*, *don't add features*, *don't make the app lie* — is preserved
verbatim.

## Modes

| Mode | Behaviour |
| --- | --- |
| **Normal** | Light cleanup. Preserves normal readability. |
| **Concise** | Removes redundancy and filler; keeps complete sentences. (Default.) |
| **Bare Mode** | Maximum compression + strict response rules. Best for coding agents and command workflows. |

TraceCompress does **not** include a joke "caveman" language mode. It never
produces broken grammar or makes a prompt less professional, and it never damages
meaning to save tokens.

## Bare Mode

Bare Mode tells the agent to stop wasting output tokens. Its response-control
block instructs the agent to: be direct, not sugarcoat, not lie, not claim
completion unless verified, not repeat the request, not paste full files, show
only changed code, state what changed / failed / remains, prefer bullets and
exact commands, and keep output minimal unless you ask for detail.

## Output budgets

Output-budget presets generate response-size guidance (this is guidance, not
enforcement, unless the provider supports hard limits):

| Preset | Guidance |
| --- | --- |
| **Tiny** | Under ~8 bullets. No paragraphs, no full files. Only changed files, commands, failures, next step. |
| **Short** | Concise bullets. No repeated explanations. Snippets only when necessary. |
| **Normal** | Balanced. Explain only non-obvious decisions. |
| **Detailed** | May explain reasoning and include relevant code. Still no filler. |

## Conflict detection

If a prompt contains contradictory instructions, TraceCompress **shows the
conflict** rather than guessing. For example, "make it minimal" plus "add a huge
animation-heavy UI" surfaces a conflict to resolve. Conflicts are never hidden.

## Local-first behaviour

Compression is a local, deterministic engine: it normalizes whitespace, removes
duplicate sentences and filler, merges repeated constraints, and preserves code
blocks, quoted strings, commands, file paths, env vars, and URLs. An external-LLM
compressor is **opt-in only** (`external_llm = true`); if enabled, TraceGuard
warns that prompt text may leave the machine and records the provider and model.

## Token estimates

Token counts are **local estimates** (a ~4-characters-per-token heuristic with a
word-count floor). They are labelled as estimates everywhere and are not exact
provider tokenizer output. An exact tokenizer can be added opt-in later.

## Privacy

Configure in `.traceguard/config.toml`:

```toml
[prompt_compression]
enabled = true
default_mode = "concise"
prompt_history = true
external_llm = false
```

When `prompt_history = false`, TraceGuard stores only token estimates, the
mode, a content hash, and metadata — **never the raw prompt text**. Stored
prompts (when enabled) live only in local SQLite and are never uploaded.

## CLI usage

```bash
# Inline
trg compress-prompt "fix the auth bug but do not rewrite the app"

# Interactive (paste, then Ctrl-D / Ctrl-Z)
trg compress-prompt

# Modes
trg compress-prompt --mode normal
trg compress-prompt --mode concise
trg compress-prompt --mode bare

# Attach an output budget
trg compress-prompt --mode bare --output-budget tiny

# Compress the agent prompt as part of a run (shows before/after, asks first)
trg run --compress "claude fix the login bug but do not rewrite the whole app"
trg run --compress --mode bare "claude add tests for src/auth/login.ts"
```

`trg run --compress` compresses, shows the before/after and estimates, asks for
confirmation, runs the agent with the compressed prompt, and stores the
compression metadata with the run.

## Dashboard usage

Open the **Prompt Compressor** page in the dashboard. Paste a prompt, pick a mode
(Normal / Concise / Bare Mode), and click **Compress**. You get the compressed
prompt, response rules, before/after token estimates, preserved constraints,
removed redundancy, and any conflicts. Use **Copy** or **Use in next run**. Build
an output-budget block from the Tiny / Short / Normal / Detailed presets.

## Limitations

- Token estimates may not be exact.
- Compression can reduce clarity if used carelessly; review the result.
- TraceCompress warns on conflicting instructions rather than resolving them.
- External-LLM compression is opt-in only.
- Output budgets are instructions unless the provider supports hard limits.
- To stay safe, a clause that contains a constraint keyword is preserved
  verbatim, so highly constraint-dense prompts compress less.
