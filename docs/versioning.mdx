The current version is **1.1**.

```bash
trg --version
# TraceGuard 1.1

traceguard --version
# TraceGuard 1.1
```

## Single source of truth

The version lives in one place ‚Äî `traceguard-core::VERSION` ‚Äî and is surfaced by:

- the `trg` / `traceguard` CLI (`--version`, `-V`)
- the daemon API (`GET /api/health` ‚Üí `version`, `GET /api/state` ‚Üí `version`)
- the local dashboard (sidebar footer: "TraceGuard v1.1")
- this documentation and the README
- the GitHub release tag (`v1.1.0`) and the Homebrew formula

## Scheme

TraceGuard uses a simple subversion scheme (`1.1`, `1.2`, ‚Ä¶). The Cargo crate
version uses the matching semver (`1.1.0`) because Cargo requires `x.y.z`, but
the product version shown to users is the short form.

The version only advances when the project owner says so.
